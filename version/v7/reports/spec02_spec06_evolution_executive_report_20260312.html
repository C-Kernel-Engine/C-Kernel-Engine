SVG Capability Evolution Executive Report

This page summarizes the capability progression from spec02 through spec06. The early runs were raw-SVG experiments with older legacy probes; the later runs use structured probe contracts and richer held-out evaluation. The numbers are not perfectly comparable across every line, but the capability trend is.

Bottom line. The line has progressed from raw valid SVG emission, to representation failure diagnosis, to structured scene control, and then into held-out infographic template filling. With spec06 r6, the work is now in very narrow repair territory, not basic feasibility territory.

Evidence Snapshots

What Changed by Line

Spec02: raw SVG generation proved the training/runtime stack could produce valid files, but the output surface was too entangled for reliable control.

Spec03: the bootstrap/tokenizer path failed despite training running, which made the representation problem impossible to ignore.

Spec04: the structured scene DSL solved syntax, closure, and renderability, and enabled automatic balanced probes. But curriculum-only tweaks inside a flat atom target were not enough to get composition and control binding.

Spec05: richer minimal-pair midtrain data plus a modest architecture increase produced the first real held-out semantic generalization on structured scenes.

Spec06: the same structured approach extended to infographic templates with deterministic layout geometry and model-predicted content slots. By r6, this line reached 91.7% overall exact and 100% renderability, which is the clearest evidence yet of practical infographic capability.

Why This Matters

  • This is not just “making SVGs.” It is building a governed benchmark for structured generation that can be scored automatically.
  • The line now demonstrates progression from syntax learning to layout control and then to richer templated infographic composition.
  • The remaining failures in spec06 are narrow and diagnosable, which is a much stronger engineering position than broad collapse.
