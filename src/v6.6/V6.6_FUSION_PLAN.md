# V6.6 Fusion Kernel Implementation Plan

## Goal

Implement fused kernels that **eliminate intermediate DRAM writes** between operations, keeping activations in L1/L2 cache. This is the core value proposition of C-Kernel-Engine.

## Success Criteria

**Before generating fused kernels in codegen, we MUST prove they are faster:**

| Metric | Requirement | How to Measure |
|--------|-------------|----------------|
| Correctness | Output matches non-fused within 1e-5 | Unit tests vs PyTorch/llama.cpp |
| Speedup | Fused kernel ≥ 1.3x faster than separate | `bench_fusion.py` |
| Cache efficiency | L3 miss rate reduced | `perf stat -e cache-misses` |

**If fused kernels are NOT faster, we do NOT ship them.** The whole point is performance.

## Architecture: Three Fusion Blocks

Based on transformer decode path analysis, there are exactly **3 places we MUST write to DRAM**:

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         DECODE TOKEN FLOW                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  Input token_id                                                         │
│       │                                                                 │
│       ▼                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ FUSION BLOCK 1: Input → KV Cache                                │   │
│  │                                                                 │   │
│  │  embed ──► RMSNorm ──► Q,K,V projection ──► RoPE(Q,K)          │   │
│  │                                                                 │   │
│  │  All intermediate activations stay in L1/L2                     │   │
│  │  Only K,V written to DRAM (KV cache) ◄── DRAM WRITE #1         │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│       │                                                                 │
│       ▼ Q (in registers), K,V (from cache)                             │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ FUSION BLOCK 2: Attention → MLP → Next Layer                    │   │
│  │                                                                 │   │
│  │  Attention(Q, K_cache, V_cache)                                 │   │
│  │       │                                                         │   │
│  │       ▼                                                         │   │
│  │  + residual ──► RMSNorm ──► MLP (gate, up, down) ──► + residual│   │
│  │       │                                                         │   │
│  │       ▼                                                         │   │
│  │  hidden_out (passed to next layer in registers)                 │   │
│  │                                                                 │   │
│  │  Zero DRAM writes! All in L1/L2/registers                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│       │                                                                 │
│       ▼ (repeat Block 1 + Block 2 for each layer)                      │
│       │                                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ FUSION BLOCK 3: Final Layer → Logits                            │   │
│  │                                                                 │   │
│  │  final_hidden ──► RMSNorm ──► lm_head projection ──► logits    │   │
│  │                                                                 │   │
│  │  logits written to DRAM ◄── DRAM WRITE #2                      │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│       │                                                                 │
│       ▼                                                                 │
│  Output: logits → sampling → next token                                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

TOTAL DRAM WRITES PER TOKEN: 2 (KV cache + logits)
vs NON-FUSED: ~50+ DRAM writes (every intermediate buffer)
```

## Implementation Order

### Phase 1: Foundation (Week 1)

#### 1.1 Create Fused Kernel Directory Structure
```
src/kernels/fused/
├── rmsnorm_qkv.c              # RMSNorm + Q,K,V projection
├── rmsnorm_qkv_rope.c         # + RoPE applied to Q,K
├── attention_output.c          # Attention + output projection + residual
├── rmsnorm_mlp.c              # RMSNorm + MLP (gate, up, down) + residual
└── rmsnorm_lmhead.c           # Final: RMSNorm + lm_head projection

tests/                          # ALL kernel tests go here (existing)
├── test_fused_rmsnorm_qkv.py  # Fused kernel correctness + benchmark
├── test_fused_rmsnorm_rope.py
├── test_fused_attention.py
├── test_fused_mlp.py
└── test_fused_lmhead.py

src/v6.6/                       # Codegen that USES fused kernels
├── codegen uses fused kernels
└── IR emits fused kernel calls
```

**Key principle**: Kernels are Lego blocks. Tests live in `tests/`, not version folders.

#### 1.2 First Kernel: `rmsnorm_qkv`

**Why start here:**
- Simplest fusion (no control flow)
- High impact (3 GEMV reads of same normed buffer)
- Easy to test (deterministic, no attention mask)

**Implementation:**
```c
// src/kernels/fused/rmsnorm_qkv.c

/**
 * Fused RMSNorm + QKV Projection
 *
 * NON-FUSED version:
 *   rmsnorm(x, weight, normed);           // normed written to DRAM
 *   gemv_q4k(wq, normed, q);              // normed read from DRAM
 *   gemv_q4k(wk, normed, k);              // normed read from DRAM
 *   gemv_q4k(wv, normed, v);              // normed read from DRAM
 *   // 4 DRAM accesses for 'normed' buffer
 *
 * FUSED version:
 *   rmsnorm_qkv_fused(x, weight, wq, wk, wv, q, k, v);
 *   // 'normed' stays in registers, 0 DRAM accesses
 *
 * Expected speedup: 1.5-2x for this operation
 */
void rmsnorm_qkv_q4k_fused(
    const float *x,           // [embed_dim] input hidden state
    const float *rms_weight,  // [embed_dim] RMSNorm weight
    const void *wq,           // [num_heads * head_dim, embed_dim] Q4_K
    const void *wk,           // [num_kv_heads * head_dim, embed_dim] Q4_K
    const void *wv,           // [num_kv_heads * head_dim, embed_dim] Q4_K
    float *q_out,             // [num_heads * head_dim]
    float *k_out,             // [num_kv_heads * head_dim]
    float *v_out,             // [num_kv_heads * head_dim]
    int embed_dim,
    int num_heads,
    int num_kv_heads,
    int head_dim,
    float eps                 // RMSNorm epsilon
);
```

import time
import statistics

class BenchFusion:
    """Benchmark fused vs non-fused kernels"""

    REQUIRED_SPEEDUP = 1.3  # Fused must be at least 30% faster

    def bench_rmsnorm_qkv(self, sizes=[(896,), (2048,), (4096,)]):
        """Benchmark RMSNorm+QKV fusion"""

        for embed_dim in sizes:
            # Warmup
            for _ in range(10):
                run_separate(x, rms_weight, wq, wk, wv)
                run_fused(x, rms_weight, wq, wk, wv)

            # Benchmark separate
            separate_times = []
            for _ in range(100):
                start = time.perf_counter_ns()
                run_separate(x, rms_weight, wq, wk, wv)
                separate_times.append(time.perf_counter_ns() - start)

            # Benchmark fused
            fused_times = []
            for _ in range(100):
                start = time.perf_counter_ns()
                run_fused(x, rms_weight, wq, wk, wv)
                fused_times.append(time.perf_counter_ns() - start)

            separate_median = statistics.median(separate_times)
            fused_median = statistics.median(fused_times)
            speedup = separate_median / fused_median

            print(f"embed_dim={embed_dim}:")
            print(f"  Separate: {separate_median/1000:.1f} µs")
            print(f"  Fused:    {fused_median/1000:.1f} µs")
            print(f"  Speedup:  {speedup:.2f}x")

            # FAIL if not faster
            assert speedup >= self.REQUIRED_SPEEDUP, \
                f"Fused kernel not fast enough! {speedup:.2f}x < {self.REQUIRED_SPEEDUP}x"

## Measurement Checklist

Before merging any fused kernel:

- [ ] **Correctness**: Passes all unit tests (vs separate, PyTorch, llama.cpp)
- [ ] **Speedup**: ≥1.3x faster than separate kernels
- [ ] **Cache efficiency**: Measurable reduction in L3 misses (`perf stat`)
- [ ] **No regressions**: Full model inference still matches reference
- [ ] **Code review**: Implementation reviewed for correctness

## Expected Results

Based on memory bandwidth analysis:

| Fusion | Non-Fused DRAM | Fused DRAM | Expected Speedup |
|--------|----------------|------------|------------------|
| RMSNorm+QKV | 4 × embed_dim | 0 | 1.5-2x |
| +RoPE | 6 × embed_dim | 0 | 1.5-2x |
| Attn+Output | 3 × embed_dim | 1 × embed_dim | 1.3-1.5x |
| RMSNorm+MLP | 6 × embed_dim | 0 | 1.5-2x |
| **Full decode** | ~50 writes | 2 writes | **2-3x** |

## Risk Mitigation

**Risk**: Fused kernels are not faster than separate
**Mitigation**:
- Benchmark FIRST before investing in codegen integration
- If <1.3x speedup, investigate why (cache behavior, instruction mix)
- Consider partial fusion (fuse only high-impact pairs)

**Risk**: Correctness issues in complex fusions
**Mitigation**:
- Test each fusion independently before combining
- Use real model weights for testing (not random)
- Compare against multiple references (PyTorch + llama.cpp)
