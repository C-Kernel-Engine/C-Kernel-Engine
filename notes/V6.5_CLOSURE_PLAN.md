## Task 1: Fix GitHub CI (Priority: BLOCKER)

### Root Cause Analysis

All 21 test failures share the same error:
```
OSError: undefined symbol: gemv_q8_0_avx2
```

**What's happening:**
- GitHub runners have AVX2 (but NOT AVX-512)
- Code is compiled with `-march=native`, so `__AVX2__` is defined
- Dispatch code calls `gemv_q8_0_avx2()` at `src/kernels/gemm_kernels_q8_0.c:390`
- But `gemv_q8_0_avx2()` is **never implemented** - only declared
- Same issue with `gemv_q5_0_avx2()` at `src/kernels/gemm_kernels_q5_0.c:383`

### Missing AVX2 Implementations

| Function | Dispatched At | Exists? | Fallback |
|----------|--------------|---------|----------|
| `gemv_q8_0_avx2` | gemm_kernels_q8_0.c:390 | **NO** | AVX or SSE |
| `gemv_q5_0_avx2` | gemm_kernels_q5_0.c:383 | **NO** | AVX exists |
| `gemv_q4_k_q8_k_avx2` | gemm_kernels_q4k_q8k.c:190 | YES | N/A |

### Fix Options

**Option A: Remove AVX2 dispatch (Quick Fix)**
Change dispatch to fall back from AVX-512 directly to AVX (skip AVX2).

```c
// BEFORE (gemm_kernels_q8_0.c:387-398)
#if defined(__AVX512F__)
    gemv_q8_0_avx512(y, W, x, M, K);
#elif defined(__AVX2__) && defined(__FMA__)
    gemv_q8_0_avx2(y, W, x, M, K);  // <- MISSING!
#elif defined(__AVX__)
    gemv_q8_0_avx(y, W, x, M, K);
...

// AFTER (skip AVX2, use AVX)
#if defined(__AVX512F__)
    gemv_q8_0_avx512(y, W, x, M, K);
#elif defined(__AVX__)
    gemv_q8_0_avx(y, W, x, M, K);  // Skip broken AVX2
#elif defined(__SSE4_1__)
    gemv_q8_0_sse(y, W, x, M, K);
...
```

**Option B: Implement AVX2 kernels (Better Performance)**
- Write actual AVX2 implementations using YMM registers
- Copy from AVX-512 pattern, use 256-bit intrinsics

### Recommended: Option A First, Then B

1. **Immediate fix** (Option A): Remove broken AVX2 dispatch → CI passes
2. **Follow-up** (Option B): Implement real AVX2 kernels for better performance on AVX2-only systems

## Reference: Q6_K Block Layout

```
block_q6_K (210 bytes for 256 weights):
├── d: FP16 (2 bytes) - super-block scale
├── ql[128]: uint8 - low 4 bits of each weight
├── qh[64]: uint8 - high 2 bits (packed 4 per byte)
└── scales[16]: int8 - per-16-weight sub-block scales

Unpacking weight j:
  low4 = (ql[j/2] >> (4 * (j%2))) & 0xF
  high2 = (qh[j/4] >> (2 * (j%4))) & 0x3
  q6 = low4 | (high2 << 4)  // 0..63
  weight = d * scales[j/16] * (q6 - 32)
```

## Reference: Q8_K Block Layout

```
block_q8_K (292 bytes for 256 values):
├── d: FP32 (4 bytes) - super-block scale (NOTE: FP32, not FP16!)
├── qs[256]: int8 - quantized values
└── bsums[16]: int16 - precomputed sums of 16 consecutive int8s

Unpacking value j:
  value = d * qs[j]
```
